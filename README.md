# globaltimeconvert.com

A fast, ad-free, cookieless timezone converter.

**Live**: <https://globaltimeconvert.com>

Searching for a timezone converter in 2026 still drops you on pages packed with ads, popups, and cookie banners. This is the opposite proposition: a quiet, fast, accurate site for the most common timezone questions, with nothing extra in the way.

## What it does

- **Current time in any city** — `/time-in-tokyo`, `/time-in-sao-paulo`, etc. Live ticking clock, locale-aware (12h with AM/PM for English, 24h for everything else), full timezone metadata, DST status, and a live comparison against ~12 nearby cities.
- **City-pair conversion** — `/tokyo-to-london` style URLs with a side-by-side live clock, an interactive time slider, and a quick-pick table of common business-hour conversions.
- **Specific-time pages** — `/9am-tokyo` or `/9am-tokyo-to-london` for natural queries like "what's 9 am in Tokyo?".
- **Six locales** — `en`, `es`, `pt`, `de`, `nl`, `zh-CN`. Other locales reuse the English city slugs but render the chrome and city names in the local language.
- **No tracking** — no third-party scripts, no cookies, no fingerprinting. The only analytics is a server-side beacon that records the path, language, country (region-anonymised), and a daily-rotating salted hash of the IP. The IP itself is never persisted.

## What it doesn't do

- Embed widgets on third-party sites.
- Email or account-based features.
- Currency conversion, weather, holidays, or anything outside the time-zone scope.
- Render an ad. Ever.

## Tech stack

- [Astro 6](https://astro.build) with the [Cloudflare adapter](https://docs.astro.build/en/guides/integrations-guide/cloudflare/) — SSR on the edge.
- [Cloudflare Workers](https://developers.cloudflare.com/workers/) for the runtime, [D1](https://developers.cloudflare.com/d1/) for the city database, [KV](https://developers.cloudflare.com/kv/) for OG-image and analytics caching, and [Workers Analytics Engine](https://developers.cloudflare.com/analytics/analytics-engine/) for the privacy-first pageview store.
- [Tailwind v4](https://tailwindcss.com) with `@theme` tokens and `oklch` colors.
- [Svelte 5](https://svelte.dev) islands (runes API) for the interactive bits — search, time slider, your-time detection.
- [GeoNames](https://www.geonames.org/) for the city dataset (CC BY 4.0).

## Local development

You'll need [Node 22+](https://nodejs.org) and [Wrangler](https://developers.cloudflare.com/workers/wrangler/) with a Cloudflare account.

```bash
git clone https://github.com/VerburgtJimmy/globaltimeconvert.com.git
cd globaltimeconvert.com
npm install

# Apply schema + seed the city data into a local D1 instance.
npm run db:migrate:local
npm run data:import          # downloads GeoNames dumps, builds data/seed.sql
npm run db:seed:local

# Start the dev server.
npm run dev
```

Then open <http://localhost:4321>.

The first `data:import` downloads ~600 MB from GeoNames (mostly `alternateNamesV2.zip` for translations); subsequent runs reuse the cached files in `data/raw/` (which is gitignored).

## Deploy your own

The site is deployed via [Wrangler](https://developers.cloudflare.com/workers/wrangler/) to Cloudflare Workers with custom-domain routing. To run your own copy:

1. Create a Cloudflare account.
2. Create the bound resources:

   ```bash
   npx wrangler d1 create globaltimeconvert
   npx wrangler kv namespace create OG_CACHE
   npx wrangler kv namespace create SESSION
   ```

3. Copy the IDs into your own `wrangler.toml`.
4. Apply the migrations and seed:

   ```bash
   npm run db:migrate:remote
   npm run data:import
   npm run db:seed:remote
   ```

5. Deploy:

   ```bash
   npm run deploy
   ```

CI/CD is wired up in `.github/workflows/` — push to `main` deploys to production, and PRs get a preview URL automatically. You'll need `CLOUDFLARE_API_TOKEN` and `CLOUDFLARE_ACCOUNT_ID` repo secrets configured.

## Project structure

```
src/
├── components/        # Astro + Svelte components
│   ├── pages/         # Page templates (CityClockPage, PairPage, etc.)
│   ├── islands/       # Svelte components hydrated client-side
│   └── seo/           # <Meta>, JSON-LD helpers
├── content/i18n/      # Locale dictionaries (one JSON per locale)
├── lib/               # tz math, db helpers, seo builders, i18n loader
├── pages/             # Astro routes — file-based routing
└── styles/            # Tailwind + global styles

data/
├── seed.sql           # Generated by `npm run data:import`
├── curated-cities.json
└── curated-translations.json

migrations/            # D1 schema migrations
scripts/               # Data-import pipeline (Node)
```
